8  데이터 과학자

데이터 과학(Data Science)은 데이터로부터 지식과 인사이트를 추출하는 학제간 융합 분야다. 통계학, 컴퓨터 과학, 도메인 전문성을 결합하여 복잡한 문제를 해결한다. 2012년 하버드 비즈니스 리뷰는 데이터 과학자를 “21세기 가장 섹시한 직업”(Davenport & Patil, 2012)이라 불렀고, 그 후 10년간 데이터 과학은 비즈니스, 과학, 정부 전 영역에서 핵심 역량으로 자리 잡았다.

그림 8.1 은 드류 콘웨이(Drew Conway)가 2010년 제시한 데이터 과학의 고전적 정의를 보여준다. 통계 및 수학 지식은 데이터의 패턴을 발견하고 불확실성을 정량화한다. 컴퓨터 과학 및 프로그래밍은 대규모 데이터를 처리하고 알고리즘을 구현한다. 도메인 전문성은 비즈니스 문제를 정의하고 결과를 해석한다. 세 영역이 교차하는 지점에 진정한 데이터 과학이 있다. 하지만 이 세 영역을 갖춘 ’데이터 과학자’라는 직업이 처음부터 존재했던 것은 아니다. 과거 30년간 데이터 관련 역할이 어떻게 진화해왔는지 추적하면, 현재 데이터 과학자의 위치와 미래 방향을 더 명확히 이해할 수 있다.

8.1 전문가 시대: 파편화된 전문가

데이터 과학이라는 직업이 등장하기 전인 1990년대부터 2000년대까지, 조직은 데이터베이스 관리자, 비즈니스 분석가, 통계학자라는 세 가지 전문가 집단으로 데이터 업무를 처리했다. 각 전문가는 특정 영역에서 탁월했지만, 다른 영역에서는 역량이 부족했다. 데이터베이스 관리자는 SQL과 데이터 관리에 전문성을 가졌지만, 비즈니스 보고와 의사소통 능력은 미약했다. 비즈니스 분석가는 엑셀을 활용한 보고와 도메인 지식에 강했으나, 수학과 통계 지식이 부족했다. 통계학자는 SAS/SPSS를 이용한 고급 통계 분석에 능했지만, 데이터베이스 기술과 의사소통 능력이 낮았다.

그림 8.2 에서 볼 수 있듯이, 이러한 전문가들은 구조화된 프로세스를 통해 협업했다. 데이터베이스 관리자가 데이터를 추출하여 처리하면, 비즈니스 분석가가 이를 검토하고, 통계학자가 최종 결과를 전달하는 방식이었다. 조직 차원에서는 비즈니스 보고, 통계 모델링, 데이터 관리라는 전문화된 역량을 갖추었지만, 프로그래밍 능력은 여전히 코볼(COBOL)이나 비주얼 베이직(Visual Basic) 수준에 머물렀다. 이는 부서별 사일로 구조로 인해 역량이 파편화되고, 통합적인 데이터 분석이 어려웠음을 의미한다.

그림 8.3 는 당시 일반적인 업무 환경에서 경험적으로 관찰되던 시간 배분 패턴을 보여준다. 데이터베이스 관리자는 대략 시간의 70% 정도를 데이터베이스 유지보수와 백업에 소비했고, 실제 분석을 위한 SQL 쿼리는 20% 내외에 불과했다. 비즈니스 분석가는 약 60%를 수작업 데이터 입력에 할애했으며, 엑셀 보고서 작성은 25% 정도, 의미 있는 분석 시간은 거의 없었다. 통계학자조차 30% 가량을 데이터 준비에 사용했다. 이러한 수치는 정확한 통계 자료가 아닌 당시 현장에서 일반적으로 경험되던 추정치이지만, 각 전문가가 자신의 핵심 역량과 무관한 영역에 시간을 빼앗기는 구조적 문제를 잘 드러낸다. 결과적으로 한 번의 분석을 완료하는 데 대략 3-4주가 소요되었다. 비즈니스 요청부터 IT 데이터 추출, 엑셀 정제, 통계 분석, 최종 보고서까지 순차적 프로세스를 거치는 동안, 실제 인사이트 도출은 전체 시간에서 극히 일부에 지나지 않았다.

8.2 팀 기반 접근: 데이터 과학 시대

2010년대 데이터 과학의 등장은 이러한 파편화를 해소하려는 시도였다. 하지만 현실에서 완벽한 데이터 과학자는 존재하지 않는다. 데이터 엔지니어는 컴퓨터 과학과 통계에 강하지만 의사소통과 도메인 전문성이 상대적으로 약하다. ML 엔지니어는 머신러닝과 수학에 탁월하지만 데이터 시각화와 도메인 지식이 부족하다. 데이터 분석가는 시각화, 의사소통, 도메인 전문성에 뛰어나지만 머신러닝과 컴퓨터 과학 역량이 낮다.

그림 8.4 가 보여주는 핵심은 명확하다. 완벽한 데이터 과학자는 없기에 팀이 필요하다. 과거 분절된 전문가들이 순차적 프로세스로 협업했다면, 현대 데이터 과학 팀은 적층 막대 구조처럼 각자의 전문성을 통합한다. 데이터 시각화는 주로 데이터 분석가가 담당하지만, ML 엔지니어와 데이터 엔지니어도 기여한다. 머신러닝은 ML 엔지니어가 주도하되, 다른 팀원들의 지원을 받는다. 이러한 적층 구조는 과거 사일로 방식과 달리, 모든 역량 영역에서 팀 전체가 중간 이상의 수준을 유지할 수 있게 한다.

그림 8.5 는 2024년 여러 산업 설문 조사를 종합한 데이터 과학 인력의 실제 시간 배분을 보여준다1. 데이터 과학자는 여전히 45% 시간을 데이터 정제에 사용하지만, 과거 80%에 비하면 자동화를 통해 크게 개선되었다. 시각화(21%), 모델링(12%), 배포(11%), 커뮤니케이션(11%)이 균형 있게 분배되어 있으며, 특히 모델링 영역에 LLM이 통합되면서 효율이 더욱 향상되었다. 데이터 엔지니어는 파이프라인 구축(35%)과 ETL 프로세스(25%)에 집중하며, 데이터베이스 관리(20%)와 최적화(15%)를 담당한다. 과거 데이터베이스 관리자가 유지보수에 70% 시간을 쓴 것과 달리, 현대 데이터 엔지니어는 가치 창출에 직접 기여하는 작업에 더 많은 시간을 할애한다. 데이터 분석가는 분석 및 인사이트 도출(30%), 보고서 작성(25%), 데이터 정제(20%)에 시간을 배분하며, 이해관계자 커뮤니케이션(15%)과 임시 요청(10%)도 처리한다. ML 엔지니어는 모델 배포 및 MLOps(30%), 모델 최적화(25%), 시스템 통합(20%)에 전문화되어 있으며, 프로덕션 환경에서의 안정성과 성능에 집중한다. 분석 엔지니어는 데이터 엔지니어링과 분석의 중간 역할로, 데이터 모델링(35%)과 파이프라인 개발(25%)을 주도하며 데이터 품질 및 테스트(20%)를 책임진다.

하단 인사이트 박스는 데이터 과학 이전 시대 대비 진화를 정량적으로 보여준다. 데이터 정제 시간은 80%에서 45%로 35% 감소했으며, 교차 기능 협업은 3배 증가하여 팀 기반 협업 문화가 정착되었다. 거버넌스 및 컴플라이언스는 0%에서 5-10%로 필수화되었고, 전문 역할 수는 3개에서 5개로 67% 증가했다. 분석 완료 주기는 3-4주에서 3-5일로 75% 단축되었으며, LLM 통합 비율은 0%에서 45%로 급증하여 AI가 데이터 과학 워크플로우의 핵심 요소로 자리 잡았다. 이러한 변화는 단순한 효율 개선을 넘어, 데이터 과학 인력의 역할과 업무 방식이 근본적으로 재구성되었음을 의미한다. GDPR, SOX, 모델 카드, AI 윤리 등 규제 요구사항이 필수화되면서, 데이터 과학 인력의 업무 범위가 기술적 역량을 넘어 법적·윤리적 책임까지 확대되었다.

8.3 강화된 협업: AI 강화 시대

2022년 ChatGPT 등장 이후, 데이터 과학은 다시 한번 진화하고 있다. AI는 단순히 코딩과 분석을 자동화하는 도구를 넘어, 팀 전체의 역량을 강화하는 촉매제가 되었다. AI 강화 데이터 엔지니어는 파이프라인 구축과 AI-ETL, 클라우드 인프라에 집중하면서 MLOps와 모니터링까지 관리한다. AI 강화 데이터 분석가는 LLM을 활용한 시각화와 자연어 쿼리로 BI 도구를 넘어선 인사이트를 생성한다. AI/프롬프트 엔지니어는 프롬프트 설계, AI 윤리, 인간-AI 협업을 전문적으로 다루는 새로운 역할로 부상했다.

그림 8.6 는 AI 시대 데이터 과학의 핵심 변화를 보여준다. AI로 강화된 팀은 향상된 워크플로우를 통해 빠르게 인사이트를 도출한다. AI 지원, 강화, 협업이라는 세 단계를 거치며, 각 역할은 기존보다 훨씬 높은 수준의 역량을 발휘한다. LLM 통합, 강화 분석, 빠른 처리, AutoML/MLOps, 자연어 처리 등 모든 영역에서 AI가 인간의 능력을 증폭시킨다. 특히 주목할 점은 AI 윤리와 데이터 전략이 새로운 핵심 역량으로 자리 잡았다는 것이다. 과거 사일로 구조의 순차적 프로세스나 데이터 과학 시대의 적층 협업과 달리, AI 시대는 모든 팀원이 AI를 통해 강화된 상태에서 동시다발적으로 협업하는 구조로 진화했다.

그림 8.7 는 2024년 이후 LLM 도입 트렌드와 플랫폼 엔지니어링 패턴을 기반으로 예측한 AI 강화 인력의 시간 배분을 보여준다. AI/ML 엔지니어는 LLM 통합(30%), 모델 최적화(25%), 프롬프트 엔지니어링(20%)에 집중하며, AI 워크플로우 자동화를 주도한다. 플랫폼 엔지니어는 플랫폼 개발(35%)과 셀프서비스 도구(30%)를 통해 데이터 민주화를 실현하며, 기술팀과 비즈니스팀 간 간극을 해소한다. 분석 엔지니어는 데이터 제품(35%)과 시맨틱 레이어(25%) 구축에 시간을 할애하며, AI 코드 어시스턴트를 활용해 파이프라인 자동화(20%)를 가속화한다. MLOps 엔지니어는 ML용 CI/CD(30%), 모델 모니터링(25%), 인프라(25%)를 통해 ML 라이프사이클 전체를 자동화하며, 모델 드리프트와 성능 저하를 실시간으로 감지한다. 데이터 제품 관리자는 전략 및 로드맵(40%), 이해관계자 관리(30%)에 집중하며, AI 기반 데이터 제품의 비즈니스 가치를 정의하고 조율한다.

가장 극적인 변화는 워크플로우 타임라인에서 확인할 수 있다. 과거 3-4주가 걸리던 분석이 이제 수 시간에서 2일로 단축되었다. 셀프서비스 요청으로 시작해 AI가 자동으로 인사이트를 생성하고, 자동 검증을 거쳐 전략적 해석을 더한 뒤 실시간 실행까지 이어지는 흐름이 확립되었다. 데이터 정제에 80%를 쓰던 과거와 달리, 이제는 AI가 대부분의 반복 작업을 처리하고 인간은 전략, 제품, 윤리와 같은 고부가가치 업무에 집중한다. 데이터 과학자에서 AI/ML 엔지니어로, 데이터 엔지니어에서 플랫폼 엔지니어로, 분석가에서 데이터 제품 관리자로 역할이 진화하면서, 데이터 조직은 단순 분석 제공자에서 비즈니스를 주도하는 전략적 파트너로 거듭났다.

8.4 데이터 과학 인력 진화와 미래

이 장에서 확인한 핵심은 명확하다. 데이터 과학 인력은 1990년대 파편화된 전문가 시대에서 2010년대 팀 기반 협업을 거쳐, 2020년대 AI 강화 시대로 진화했다. 각 시대마다 역할이 근본적으로 재정의되었고, 업무 방식과 시간 배분이 완전히 달라졌다. 데이터 정제에 80%를 쓰던 시대에서 45%로, 다시 AI 자동화를 통해 최소화된 시대로 넘어오면서, 인간의 역할은 반복 작업 처리자에서 전략적 의사결정자로 전환되었다.

역사는 중요한 패턴을 보여준다. 데이터베이스 관리자, 비즈니스 분석가, 통계학자라는 세 전문가는 데이터 엔지니어, 데이터 분석가, ML 엔지니어라는 다섯 역할로 분화되었고, 이제 AI/ML 엔지니어, 플랫폼 엔지니어, 분석 엔지니어, MLOps 엔지니어, 데이터 제품 관리자로 재편되고 있다. 단순한 역할 추가가 아니라, 기술 중심으로 조직과 업무 프로세스 전체가 재구성되는 과정이다. 전기화가 공장 레이아웃을 완전히 바꾸었듯이, AI는 데이터 조직 구조와 문화를 근본부터 변화시키고 있다. 분석 완료 주기가 3-4주에서 3-5일로, 다시 수 시간으로 단축되면서, 데이터 조직은 사후 보고자에서 실시간 의사결정 지원자로 거듭났다.

현재 AI 도입 초기 단계에 서 있다. LLM 통합 비율이 45%에 도달했지만, 진정한 혁명은 이제 시작이다. 중요한 것은 새로운 역할 이름을 아는 것이 아니라, 각자의 조직에서 AI를 어떻게 활용해 데이터 팀을 재구성할 것인가를 실천하는 것이다. 다음 장에서는 이러한 진화를 가능하게 한 핵심 도구인 통합 개발 환경(IDE)과 AI 코딩 어시스턴트를 살펴보며, 실제로 어떻게 생산성을 극대화할 수 있는지 구체적으로 다룬다.
